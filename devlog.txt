Implementation Details:

1. Dependencies Added (pom.xml):
   - Added spring-boot-starter-data-mongodb for MongoDB integration

2. Configuration (application.properties):
   - spring.data.mongodb.uri=mongodb://localhost:27017/ecommerce
   - spring.data.mongodb.database=ecommerce

3. Data Model (Product.java):
   - Created Product entity with @Document annotation
   - Fields: id, name, description, price, category, imageUrl, stockQuantity
   - Full getters/setters and constructors

4. Repository Layer (ProductRepository.java):
   - Extended MongoRepository<Product, String>
   - Custom query methods: findByCategory, findByNameContainingIgnoreCase, etc.

5. Service Layer (ProductService.java):
   - Business logic layer
   - @PostConstruct method initializes sample data on first run
   - Added 8 sample products matching existing images

6. Controller Updates (ApiController.java):
   - Injected ProductRepository
   - GET /api/products - returns all products from database
   - POST /api/products - creates new product
   - GET /api/products/category/{category} - filter by category

Testing Instructions:
1. Ensure MongoDB is running locally on port 27017
2. Run: mvn spring-boot:run (or use IDE)
3. Test endpoints:
   - GET http://localhost:8080/api/products (should return sample products)
   - POST http://localhost:8080/api/products with JSON body to add products
4. Check MongoDB Compass to verify data persistence

Notes:
- Sample data loads automatically on first startup if database is empty
- Application will fail to start if MongoDB is not running
- Update connection string in application.properties for different MongoDB setups
